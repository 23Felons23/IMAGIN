# Plan 03-02: Topic-Driven Extractor & Output Serialization

## Objective
Build the topic-driven highlight extractor (user provides a text prompt, LLM finds the most relevant clips matching that topic) and implement the dual output serialization: `highlights.json` for Remotion and `highlights.md` for human review.

## Tasks

<task>
<read_first>
- backend/highlight_extractor.py
- backend/llm_client.py
</read_first>
<action>
Add `extract_highlights_topic(transcript_path, topic_prompt, top_n=3, window_seconds=300)` to `backend/highlight_extractor.py`.

Implementation:
1. Load and chunk transcript using existing `chunk_transcript()` (reuse from plan 03-01)
2. For each chunk, send LLM a prompt:
   - System: "You are an expert podcast editor. Given a transcript segment with timestamps, find the most relevant contiguous moment that discusses the following topic: '{topic_prompt}'. Return JSON: {\"start\": float, \"end\": float, \"score\": float (0-1 relevance), \"reason\": str}. If the topic is not mentioned in this segment, return score: 0."
   - User: formatted transcript chunk with second-precision timestamps
3. Parse JSON response, validate start/end are within chunk boundaries, skip score=0 returns
4. Return top N results sorted by relevance score descending
</action>
<acceptance_criteria>
- Topic search correctly returns only clips where the topic is actually discussed (score > 0)
- Clips returned are always within the window they were found — no timeline jumps
- Gracefully handles LLM returning score 0 (topic not in this window)
</acceptance_criteria>
</task>

<task>
<read_first>
- backend/highlight_extractor.py
- .planning/phases/03-ai-highlight-extraction-llm/03-CONTEXT.md
</read_first>
<action>
Create `backend/highlight_serializer.py` with two output functions:

1. `save_highlights_json(highlights, output_path)`:
   - Writes `highlights.json` with array: `[{start, end, score, reason, mode}]`
   - `mode` field is either `"multimodal"` or `"topic"` (passed as param)

2. `save_highlights_markdown(highlights, output_path, mode, topic_prompt=None)`:
   - Writes `highlights.md` with a human-readable report:
     ```markdown
     # Highlight Report
     Mode: multimodal | topic: "..."
     Generated: [timestamp]

     ## Clip 1 — 00:04:32 → 00:06:15 (score: 0.87)
     > "Why this was selected..." 

     ## Clip 2 — ...
     ```
   - Formats timestamps as `HH:MM:SS`

Integrate both into `main.py`'s background pipeline: after `transcribe_and_diarize()` completes, call the appropriate extractor based on `job["mode"]` and save both files to `.tmp_processing/{job_id}/`.
Update job status to `"highlights_ready"` when complete.
</action>
<acceptance_criteria>
- `highlights.json` and `highlights.md` are written to the job's temp directory
- `main.py` passes `job["mode"]` and `job["topic"]` correctly to the extractor
- Job status updates to `"highlights_ready"` after serialization
</acceptance_criteria>
</task>
