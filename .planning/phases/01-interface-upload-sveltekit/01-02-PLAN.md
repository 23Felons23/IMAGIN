# Plan 01-02: State Management & Mock Backend Status

## Objective
Implement a mock state tracker for uploaded jobs so the user can view processing status and "download" the final rendered MP4s, satisfying UI-04 and UI-05.

## Tasks

<task>
<read_first>
- frontend/src/App.svelte
</read_first>
<action>
Create a component state array `jobs = []` inside the script tag of `frontend/src/App.svelte`.
When a file is dropped in the dropzone (in the `on:drop` handler), push a new job object to the array with `{ id: Date.now(), filename: file.name, status: 'Processing...', progress: 0 }`.
Implement a simulated progress timer using `setInterval` that incrementally updates the progress of the active job to 100% over 5 seconds, then changes status to 'Complete'.
</action>
<acceptance_criteria>
- `frontend/src/App.svelte` contains an array state for jobs.
- Dropping a file updates the UI to show the new processing job.
</acceptance_criteria>
</task>

<task>
<read_first>
- frontend/src/App.svelte
</read_first>
<action>
Render the `jobs` array as a brutalist "Processing Queue" section below the dropzone.
Display the filename and a high-contrast loading bar (e.g., solid black bar filling up a neon background border).
For jobs with status 'Complete', render a bold "DOWNLOAD MP4" button that triggers a browser native alert("Video Ready!").
</action>
<acceptance_criteria>
- `frontend/src/App.svelte` contains an `{#each jobs as job}` block.
- A "DOWNLOAD MP4" button exists conditionally for completed jobs.
</acceptance_criteria>
</task>

## Verification
- Drop a file into the dropzone.
- Verify a new job record appears in the queue.
- Verify the progress bar visually increments.
- Verify the "DOWNLOAD" button appears when progress hits 100%.

## must_haves
- M1: Dropping a file visualizes the job in a queue list.
- M2: Mock progress simulates backend wait time.
- M3: Completed jobs show a download button block.
