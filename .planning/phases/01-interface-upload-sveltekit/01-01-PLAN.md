# Plan 01-01: Setup Svelte & Brutalist UI Layout

## Objective
Initialize the Vite+Svelte app and implement the core high-end Brutalist UI layout, the drag-and-drop file uploader, and the configuration toggles using vanilla CSS based on `frontend-design` principles.

## Tasks

<task>
<read_first>
- .planning/PROJECT.md
</read_first>
<action>
Execute `npx -y create-vite@latest frontend --template svelte` in the project root to scaffold the app.
Navigate into `frontend` and run `npm install`.
Add base vanilla CSS styling to `frontend/src/app.css` featuring brutalist tokens (thick borders, high contrast colors like #FF5500, monospaced or ultra-bold typography).
Clear the default boilerplate in `frontend/src/App.svelte` and `frontend/src/main.js`.
</action>
<acceptance_criteria>
- `frontend/package.json` contains `svelte`
- `frontend/src/app.css` contains `:root` variables for brutalist colors and border thickness
</acceptance_criteria>
</task>

<task>
<read_first>
- frontend/src/app.css
- frontend/src/App.svelte
</read_first>
<action>
Implement `frontend/src/App.svelte`. Create a massive brutalist header "PODCAST PIPELINE".
Implement a large HTML File drag-and-drop zone using `on:drop` and `on:dragover` events. Add a state variable to hold dropped files.
Add radio buttons for "Mode: Multimodal vs Topic-Driven" with very loud focus and selected states (e.g. inverted colors).
Add a text input for the topic prompt that conditionally renders when "Topic-Driven" is selected. Apply heavy CSS box-shadow offsets to match brutalism.
</action>
<acceptance_criteria>
- `frontend/src/App.svelte` contains `on:drop`
- `frontend/src/App.svelte` contains `<input type="text"` for topic
- UI applies brutalist CSS styles directly or via `<style>`
</acceptance_criteria>
</task>

## Verification
- Run `npm run dev` in `frontend` directory and open in browser.
- Verify UI matches distinctive brutalist aesthetic.
- Verify topic input conditionally renders correctly.
- Verify file drop captures the File object in state.

## must_haves
- M1: The Svelte app runs without errors locally.
- M2: The drag-and-drop zone correctly accepts video files.
- M3: The topic input dynamically shows/hides based on the selected mode.
