# Plan 04-01: Remotion Project Setup & Vertical Layout Component

## Objective
Scaffold the Remotion React project in `remotion/`, create the 9:16 vertical canvas composition, implement the `VideoScene` component that switches camera sources based on `render_config.json`, and setup the root composition.

<task>
<read_first>
- remotion/package.json
</read_first>
<action>
Create `remotion/src/Root.tsx` — the Remotion composition root:
```tsx
import {Composition} from 'remotion';
import {PodcastClip} from './PodcastClip';

export const RemotionRoot = () => (
  <Composition
    id="PodcastClip"
    component={PodcastClip}
    durationInFrames={1800}  // overridden at render time
    fps={30}
    width={1080}
    height={1920}
    defaultProps={{
      renderConfig: null,
    }}
  />
);
```

Create `remotion/src/PodcastClip.tsx` — the main composition component:
- Reads a `renderConfig` prop containing: `{clipStart, clipEnd, videoFiles: {[speaker]: string}, words: [{word, start, end, speaker}], emojis: {[word]: string}}`  
- Uses Remotion `<OffthreadVideo>` for video sources (better performance than `<Video>`)
- Calculates which video file should be shown at any given frame based on the word's speaker at that timestamp — using `useCurrentFrame()` and `useVideoConfig()`
- Shows the correct camera by rendering only the matching `<OffthreadVideo>` at full 1080×1920, cropping/covering to fill vertically
</action>
<acceptance_criteria>
- `remotion/src/PodcastClip.tsx` renders without TypeScript errors
- Camera switching logic uses `useCurrentFrame()` to determine active speaker at each frame
</acceptance_criteria>
</task>
