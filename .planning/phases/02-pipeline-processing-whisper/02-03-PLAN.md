# Plan 02-03: Transcription and Speaker Diarization (WhisperX)

## Objective
Pass the newly aligned master audio through a WhisperX container script to isolate speakers and return an intricate JSON map of every transcribed word with hard timestamps.

## Tasks

<task>
<read_first>
- backend/main.py
</read_first>
<action>
Add `whisperx` alongside generic ML ecosystem variables to `backend/requirements.txt` (or separate script).
Create `backend/transcriber.py`. Implement `transcribe_and_diarize(audio_path, hf_token)`.
Programmatically load the WhisperX model pipeline `whisperx.load_model("base", device="cpu", compute_type="int8")` or cuda appropriately.
Run transcriber, then load the diarization pipeline `whisperx.DiarizationPipeline(use_auth_token=hf_token)` and assign speakers to the transcribed segments natively.
</action>
<acceptance_criteria>
- The file exposes an async or callable structure yielding final structured object dictionaries.
- Handles the pyannote token correctly.
</acceptance_criteria>
</task>
