# Plan 02-02: Multicam Audio Extraction & Cross-Correlation Sync

## Objective
Implement background processes in Python to extract audio identically from video files via `ffmpeg`, then align multiple camera feeds automatically using SciPy's fft cross-correlation algorithms.

## Tasks

<task>
<read_first>
- backend/main.py
</read_first>
<action>
Update `backend/requirements.txt` to include `scipy` and `numpy`.
Create `backend/audio_sync.py` and write a robust subprocess wrapper function `extract_audio(video_path, output_wav_path)`.
This wrapper must execute `ffmpeg -y -i <video> -ac 1 -ar 16000 -vn <output>` effectively extracting monophonic 16kHz audio optimal for correlation sequences.
</action>
<acceptance_criteria>
- `backend/audio_sync.py` abstracts FFmpeg extraction correctly.
- Outputs are forced to 16kHz audio for numerical stability.
</acceptance_criteria>
</task>

<task>
<read_first>
- backend/audio_sync.py
</read_first>
<action>
Implement `find_audio_offset(master_wav, secondary_wav)` in `backend/audio_sync.py`.
Load the WAV numerical arrays using `scipy.io.wavfile.read`.
Derive the signal lag by determining the max peak resulting from `scipy.signal.fftconvolve(secondary_wav[::-1], master_wav, mode='full')` or similar robust offset math.
Return the offset shift in seconds.
Inject this logic into the background processing task pipeline in `main.py` if multiple cameras are detected per job.
</action>
<acceptance_criteria>
- Offset math reliably returns precision delay seconds without OOM crashes.
- Unnecessary audio tracks are ignored once aligned, promoting a single Master audio track.
</acceptance_criteria>
</task>
