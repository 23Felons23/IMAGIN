# Plan 02-01: FastAPI Backend & Temporary File Management

## Objective
Establish the FastAPI application base, configure CORS for the SvelteKit frontend, implement the file upload routes, and create the temporary `.tmp_processing` storage wrapper.

## Tasks

<task>
<read_first>
- .planning/PROJECT.md
</read_first>
<action>
Create a new directory `backend` at the project root.
Initialize a Python `requirements.txt` with `fastapi`, `uvicorn`, `python-multipart`, and `aiofiles`.
Create `backend/main.py` scaffolding a basic FastAPI backend.
Configure `CORSMiddleware` to allow all origins so SvelteKit locally can proxy to it safely.
</action>
<acceptance_criteria>
- `backend/requirements.txt` exists.
- `backend/main.py` can be executed to serve an API.
</acceptance_criteria>
</task>

<task>
<read_first>
- backend/main.py
</read_first>
<action>
Implement the primary POST `/api/upload` endpoint in `main.py` to accept `multipart/form-data`.
Generate a UUID for the `job_id`. 
Construct a local folder `.tmp_processing/{job_id}` securely.
Save the incoming video byte fragments natively into that directory as `camera1.mp4` by chunking `await file.read(1024*1024)`.
Return the `job_id` and initial processing status synchronously to SvelteKit, while triggering a mock `BackgroundTasks` hook.
</action>
<acceptance_criteria>
- The `/api/upload` route buffers chunked files safely instead of blowing up memory.
- The `.tmp_processing` namespace accurately isolates jobs.
</acceptance_criteria>
</task>

## Verification
- Start the API.
- Fire a cURL command manually pushing a test video.
- Ensure the isolated job folder structures spawn under `.tmp_processing`.
